# Handwritten Notes To Markdown

本 skill 服务“手写学习/推导笔记图片转 Markdown”。它不是通用 OCR、扫描论文识别、版面复刻或自动推导补全工具。

## 默认原则

- 默认执行忠实转写，不输出计划，不等待确认；只有页序不明、将覆盖已有文件、阶段边界不清、图片质量严重不足或输入超过 30 页时先询问用户。
- 默认输出中文 Markdown；原稿中的英文术语、英文句子和数学符号保持原样，不主动翻译。
- 默认生成单个忠实转写稿；用户明确要求“整理、润色、教学版、补全逻辑”时，另生成整理版文件，不和忠实转写稿混在同一文件。
- 默认不在正文写入原图文件名、原图路径，也不嵌入图片。正文只保留页边界：`## 页 1`、`## 页 2`。
- 可以查阅项目 `CONTEXT.md`、同目录笔记或用户给的参考文档辅助识别术语，但只能用于术语校对，不得把参考文档内容补写进忠实转写稿。

## 页序与输出位置

- 页序优先使用用户给出的顺序。
- 用户未说明顺序时，按文件名中的数字做自然排序。
- 文件名无法判断时，按文件修改时间排序。
- 仍有歧义时，先询问用户确认页序。
- 默认把输出文件放在图片目录的上一级；用户指定输出路径时按用户指定。
- 若目标文件已存在，先询问用户，不要直接覆盖。
- 默认命名：
  - 忠实转写稿：`主题_手稿图片忠实转写.md`
  - 整理版：`主题_手稿图片整理版.md`
  - 用户未给主题时，用图片目录名或父目录名生成主题。

## 忠实转写规则

- 清楚可读的文字正常转写。
- 明显划掉且不影响语义的内容不转写。
- 划掉但对理解有价值的内容写为 `【划掉：...】`。
- 看不清的内容写为 `【辨认不清】`。
- 根据上下文能猜到但不确定的内容写为 `【疑似：...】`，不得直接写成确定正文。
- 公式符号不确定时写为 `【符号疑似：...】`，并保留该公式所在位置。
- 不为了通顺而补写、纠错、统一符号或替换作者原意。
- 原稿前后符号不一致时，在页内备注说明“原稿此处符号可能与前文不一致”，不要在忠实转写稿中擅自统一。

## 公式规则

- 清楚的行内公式使用 `$...$`。
- 独立公式使用 `$$...$$`。
- 矩阵尽量使用 `bmatrix`。
- 忠实转写阶段只做轻度 LaTeX 化：把清楚公式转成可读 LaTeX，但不补齐缺失维度、不自动改写成标准符号。
- 整理版才允许强 LaTeX 化、统一符号、补充解释和重排推导逻辑，并必须明确这是二次整理。

## 图形、批注与框选

- 箭头转成语义文字或 LaTeX 箭头，例如 `因此：`、`$$\Downarrow$$`、`$A \rightarrow B$`。
- 旁注、圈注写成引用块，例如 `> 旁注：...`。
- 简单流程图可转成项目符号；只有用户明确要求时才改画 Mermaid。
- 复杂草图在忠实转写阶段只描述其语义，例如“原稿此处画了闭环框图，表示 ...”，不要追求版面复刻。
- 框选内容必须显式标出边界：

```md
> 【框选开始】
>
> 框选内容……
>
> 【框选结束】
```

## 忠实转写稿模板

```md
# 手稿图片忠实转写

> 说明：
> - 本稿按图片页序忠实转写。
> - `【辨认不清】` 表示无法可靠识别。
> - `【疑似：...】` 表示根据上下文推测但不确定。
> - 本阶段不补写、不统一符号、不重构推导。

---

## 页 1

...

---

## 页 2

...
```

只有存在疑似或看不清内容时，在文件末尾追加：

```md
## 待校对项

- 页 N：说明待校对内容。
```

## 整理版规则

- 只有用户明确要求整理版时才生成。
- 整理版必须另存文件，不能覆盖或替代忠实转写稿。
- 整理版可以统一符号、补全跳步、重排推导、添加解释，但必须基于忠实转写稿，并避免声称新增内容来自原稿。

## 自检与最终回复

完成后必须自检：

- 输出页标题数量是否等于输入图片数量。
- 页序是否符合用户顺序或自然排序规则。
- 所有不确定内容是否显式标为 `【疑似：...】`、`【辨认不清】` 或 `【符号疑似：...】`。
- 忠实转写稿是否没有补写原稿未出现的推导结论。
- 独立公式是否使用 `$$...$$`，矩阵是否尽量使用 `bmatrix`。
- 是否只生成了用户要求的产物，整理版是否单独成文件。

最终回复只给短报告，不粘贴全文。报告输出文件路径、输入图片数、输出页数、自检结果、主要不确定项位置，以及未做事项（例如未生成整理版、未嵌图）。
